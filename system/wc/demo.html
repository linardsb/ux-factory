Web Component wrappers — plain-page check

This page loads only the two token CSS layers (contract + neutral pack) and the wrapper modules from /system/wc/ — no components.css, no site chrome, no framework. Everything below is styled by the wrappers' own shadow-DOM CSS, themed through the token contract via custom-property inheritance.

Verdant is a fictional product, invented for this demonstration. No real company, users, or data are involved.

1 · Every spec state, set via attributes

vd-status-chip: ok · due · overdue

vd-plant-card: default (no photo → monogram) · due · overdue · no species (line omitted)

vd-care-task-row: default · due · overdue · checked

2 · DataContract-shaped JSON, via the data property

The elements below are rendered by assigning the sample records from the specs' ## Data binding sections to el.data — the same records the contracts in the handoff pack validate. The plant sample's photoUrl normally points at a scenario asset; here it is swapped for an inline SVG data URI (still contract-valid) so this page ships no image files. The second card omits photoUrl and species → monogram placeholder, compacted card.

3 · Re-skin strip — token overrides pierce the shadow DOM

The same three elements inside a wrapper <div> that overrides four semantic tokens inline. Nothing else changes — custom properties inherit through the shadow boundary, so a consumer's pack (or any scoped override) is the whole theming story.

4 · Composed events

vd-toggle and vd-select are plain CustomEvents with bubbles + composed — they cross the shadow boundary; this log listens on document. Click any row or card above.
