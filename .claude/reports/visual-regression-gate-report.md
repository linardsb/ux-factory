## Deviations from the plan
1. **Scope expanded to 8 pages / `Part of #9`** (the plan's flagged `★` user decision). The plan defaulted to 6 IA pages → `Closes #9`; **the user chose at execution time to pull the two data-connected proto pages into v1**, making this `Part of #9` (closure waits on the deferred `derive`/`agentic`/`trace` interactive surfaces). This drove deviations 2–5. Baseline count: **16, not 12**.
2. **`beforeEach` hermetic block tightened from "allow all 127.0.0.1" to "allow only 127.0.0.1:4757"** (the static server). This explicitly aborts the mock Worker on `127.0.0.1:8787` → the proto pages deterministically fall back to committed static fixtures (`source: "static"`) with an immediate abort (no 2.5s timeout wait), in *any* environment. **No-op for the IA pages** — they never request `:8787`.
3. **Per-page-kind settle strategy.** Proto pages have no `site.js` chrome, so they wait on `#source[data-source="static"]` (the settled fallback indicator, set synchronously just before render) + a data-bound rows selector (`.vd-plant-card` / `.fw-lane`). This is self-guarding: if the fallback logic were wrong, the page renders the `catch`-branch error text, `#source` never gets `data-source`, and generation *fails loudly* rather than producing a bad baseline.
4. **Proto capture pins the `dvh` phone screen.** Verdant's `.proto-screen` is `height: min(800px, 80dvh)` — the only viewport-height dependency across the 8 pages. Before measuring content height, proto captures set a tall viewport (1600px) so the frame hits its 800px cap and stops depending on height; the measure→resize→capture flow is then byte-stable (verdant's content is always > 1000px, so the cap holds after resizing to exact content height). IA pages and fieldwork have no `dvh` sizing → this step is a no-op for them. Confirmed by the two-run identical compare on verdant.
5. **Baselines regenerated fresh in Docker, not reused from the planning spike.** The spike's working-tree baselines had staggered timestamps straddling the `fullPage → integer-viewport` fix and predated the proto pages; all 16 were regenerated from the final committed spec to guarantee they're canonical and Linux-rendered.
6. **Added `permissions: contents: read` to the `visual` job** (the plan's YAML omitted it). This matches the least-privilege pattern PR #31 established on the `verify` job; `contents: read` covers checkout + test + artifact upload.
7. **Kept `package.json`'s `update:docker` without `--ipc=host`.** The plan's `package.json` IMPLEMENT block included it, but the plan's own Level 2/3 validation commands and the validated reference omit it — and CI's `container:` can't set it anyway, so omitting it locally makes generation conditions mirror CI. (If chromium ever OOMs on `/dev/shm`, the documented fix is `launchOptions.args: ['--disable-dev-shm-usage']`, added reactively — not pre-emptively.)
