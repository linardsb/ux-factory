## Deviations from the plan
1. **Tool fence — `tools` vs `allowedTools` (safety-critical correction).** The plan's literal `allowedTools: ['Read','Grep','Glob','Write','Edit','Bash']` would have **bypassed the write-fence**: SDK 0.1.77's own `.d.ts` says `allowedTools` = "auto-allowed **without prompting**" — i.e. `canUseTool` is *not* called for those tools. Implemented the intent correctly: `tools` = the six (availability), `allowedTools` = the three read-only (auto-approve), and `canUseTool` fences Write/Edit (repo + secrets) and Bash (`node …` only). This is exactly the "validate task sanity" the plan asked for.
2. **Recorder marker detection loosened (deviation from the literal regex; honors the prose contract).** The plan pinned `/^\s*\[\[piv:…\]\]/` (anchored to the block *start*). Real runs showed models emit conversational preamble before the marker and sometimes cram two markers into one text block, so a start-anchored scan silently *missed* markers. Changed to "marker **alone on any line**" (`/^[ \t]*\[\[piv:…\]\][ \t]*$/gm`) + record every marker per block, adopt the last as current phase. This matches the plan's own prose ("marker alone on its own line") and is what made a reliably clean four-act trace possible. The curator's strip regex uses the same family; the validator's `[[piv:` remnant check is unchanged.
3. **`PIV_SYSTEM` prompt iterated (spike-5 lever #2, expected).** Rewrote it as a literal numbered protocol (plan-marker before any tool call; plan states intent only; gate is a mandatory review-only block; `[[piv:implement]]` immediately precedes the Write) after diagnosing each dirty run from its raw trace. That tighten-and-re-run loop **is** spike 5; it never touched trace content. Full log in AMENDMENTS.
4. **`--dry` fence root `realpathSync`'d.** macOS `os.tmpdir()` is `/var/folders/…` but the real path is `/private/var/folders/…`; without `realpathSync` the write-fence `startsWith` would deny the agent's dry writes and look like a model failure. (Advisor-flagged; not in the plan.)
5. **Model = `claude-sonnet-5` for the committed exhibit (within plan assumption #3).** Tried `claude-opus-4-8` once at the user's request; it emitted all four markers but crammed gate+implement into one block (an *empty gate act* on screen) and cost 2.5×. sonnet-5 separates the acts cleanly and reads as engineering. Meta records `claude-sonnet-5` — honest either way.
6. **Curator keeps original `seq` values (gaps allowed) rather than re-sequencing** dropped steps, preserving raw↔curated correspondence for the diff story. Validator requires strictly-increasing seq (gaps fine).
7. **Display-time path relativization (advisor-flagged polish).** The committed JSONL carries absolute local paths (`/Users/Berzins/…`) in tool-input/response fields — honest (it's what ran) but unpolished for a public "verify my craft" exhibit. Added a `relativize(s, cwd)` transform applied at **display** time in the player (tool hints, responses, errors) against `meta.cwd`; the raw JSONL stays verbatim (the inspectable anchor — this is presentation, the same class as response truncation, and recording-time relativization would misrepresent the real call). The agent's narration is never transformed (it carries no absolute paths). Verified: 0 home-dir paths render on the main trace; the raw/curated files are unchanged and still validate.

No hand-written or hand-edited trace content anywhere — recorder + curator are the only producers (verified by the honesty diff). Path relativization is a display transform only; it never touches the committed files.
