## Issues encountered
- **Un-hide regex bug, found + fixed during local verification.** My first `data-when="real"` un-hide regex ended `…"real"\b`; the `\b` after the closing quote requires a word/non-word boundary, but the quote is followed by a space (both non-word), so the match silently failed and the real trace-note kept its `hidden` attribute (would render invisible on a real instance). The independent residue grep (advisor's step 2) caught it. Fixed by dropping the trailing `\b`, and added a validator assertion (`no element keeps a standalone hidden attribute`, `<style>` stripped so CSS `overflow: hidden` doesn't false-trip) so it can't regress silently.
- **Throwaway spike artifacts live only in the scratchpad** (`…/scratchpad/harborlight/`, `…/scratchpad/inst-out/`), never in the repo. `git status` confirms no `harborlight`/`inst-out`/`tokens.harborlight` under version control.
